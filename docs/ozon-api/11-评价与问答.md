# 评价与问答

> Ozon Seller API — 评价与问答
>
> 基础地址：`https://api-seller.ozon.ru`
>
> 认证请求头：`Client-Id`、`Api-Key`、`Content-Type: application/json`

**评价管理**（8）

- `POST` `/v1/review/comment/create` — 对评价留下评论

- `POST` `/v1/review/list` — 获取评价列表
- `POST` `/v2/review/list` — 获取评价列表

**问题和回答管理**（8）

- `POST` `/v1/question/answer/create` — 创建对问题的回答
- `POST` `/v1/question/answer/delete` — 删除问题回答
- `POST` `/v1/question/answer/list` — 问题回答列表
- `POST` `/v1/question/change_status` — 更改问题状态

- `POST` `/v1/question/info` — 问题详情
- `POST` `/v1/question/list` — 问题列表

**标签代码管理**（5）

- `POST` `/v6/fbs/posting/product/exemplar/set` — 检查并保存份数数据
- `POST` `/v6/fbs/posting/product/exemplar/create-or-get` — 获取已创建样件数据
- `POST` `/v5/fbs/posting/product/exemplar/status` — 获取样件添加状态
- `POST` `/v5/fbs/posting/product/exemplar/validate` — 标志代码验证
- `POST` `/v1/fbs/posting/product/exemplar/update` — Обновить данные экземпляров

## 评价管理

### `POST` `/v1/review/comment/create`

**对评价留下评论**

`operationId: ReviewAPI_CommentCreate`

仅适用于拥有 [评价管理](https://global-help.ozon.com/zh/work-with-customers/managing-reviews/managing-reviews-subscription/) 或 Premium Pro 订阅的卖家。  您可以在开发者社区 Ozon for dev 的[讨论](https://dev.ozon.ru/community/1190-Metody-dlia-raboty-s-otzyvami)区中，留下对此方法的反馈。

#### 请求体（`v1CommentCreateRequest`）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| mark_review_as_processed | boolean | 否 | 更新评论状态： - `true` — 状态将变更为 `Processed`（已处理）； - `false` — 状态不变。 |
| parent_comment_id | string | 否 | 父级评论的标识符（您要回复的评论）。 |
| review_id | string | 是 | 评价标识符。 |
| text | string | 是 | 评论内容。 |

#### 响应（`v1CommentCreateResponse`）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| comment_id | string | 否 | 评论标识符。 |

### `POST` `/v1/review/comment/list`

**评价的评论列表**

`operationId: ReviewAPI_CommentList`

仅适用于拥有 [评价管理](https://global-help.ozon.com/zh/work-with-customers/managing-reviews/managing-reviews-subscription/) 或 Premium Pro 订阅的卖家。  您可以在开发者社区 Ozon for dev 的[讨论](https://dev.ozon.ru/community/1190-Metody-dlia-raboty-s-otzyvami)区中，留下对此方法的反馈。  该方法返回已通过审核的评价评论信息。

#### 请求体（`v1CommentListRequest`）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| limit | integer | 是 | 限制回复中的值数量。 最少 — 20；最多 — 100。 |
| offset | integer | 否 | 从列表开头跳过的元素数量：例如，如果 `offset = 10`，那么回复将从找到的第11个元素开始。 |
| review_id | string | 是 | 评价标识符。 |
| sort_dir | string | 否 | 排序方向：   - `ASC` — 按升序。   - `DESC` — 按降序。 |

#### 响应（`v1CommentListResponse`）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| comments | array<object> | 否 | 评论信息。 |
|   comments[].id | string | 否 | 评论标识符。 |
|   comments[].is_official | boolean | 否 | `true`：评论是由官方人员留下的；`false`：评论是由买家留下的。 |
|   comments[].is_owner | boolean | 否 | `true`：评论是由卖家留下的；`false`：评论是由买家留下的。 |
|   comments[].parent_comment_id | string | 否 | 父级评论的标识符（需要对此评论进行回复）。 |
|   comments[].published_at | string | 否 | 评论发布日期。 |
|   comments[].text | string | 否 | 评论内容。 |
| offset | integer | 否 | 搜索结果中的元素数量。 |
