# 促销与定价策略

> Ozon Seller API — 促销与定价策略
>
> 基础地址：`https://api-seller.ozon.ru`
>
> 认证请求头：`Client-Id`、`Api-Key`、`Content-Type: application/json`

---

## 目录

**Ozon促销活动**（8）

- `GET` `/v1/actions` — 活动清单
- `POST` `/v1/actions/candidates` — 可用的促销商品清单
- `POST` `/v1/actions/products` — 参与 活动的商品列表
- `POST` `/v1/actions/products/activate` — 在促销活动中增加一个商品
- `POST` `/v1/actions/products/deactivate` — 从活动中删除商品
- `POST` `/v1/actions/discounts-task/list` — 申请折扣列表
- `POST` `/v1/actions/discounts-task/approve` — 同意折扣申请
- `POST` `/v1/actions/discounts-task/decline` — 取消折扣申请

**定价策略**（12）

- `POST` `/v1/pricing-strategy/competitors/list` — 竞争对手名单
- `POST` `/v1/pricing-strategy/list` — 策略列表
- `POST` `/v1/pricing-strategy/create` — 创建策略
- `POST` `/v1/pricing-strategy/info` — 策略信息
- `POST` `/v1/pricing-strategy/update` — 更新策略
- `POST` `/v1/pricing-strategy/products/add` — 将商品添加到策略
- `POST` `/v1/pricing-strategy/strategy-ids-by-product-ids` — 策略ID列表
- `POST` `/v1/pricing-strategy/products/list` — 策略中的商品列表
- `POST` `/v1/pricing-strategy/product/info` — 竞争对手  的商品价格
- `POST` `/v1/pricing-strategy/products/delete` — 从策略中删除商品
- `POST` `/v1/pricing-strategy/status` — 更改策略状态
- `POST` `/v1/pricing-strategy/delete` — 删除策略

**Ozon促销活动**（4）

- `POST` `/v1/actions/auto-add/products/list` — 获取促销活动自动添加列表中的商品列表
- `POST` `/v1/actions/auto-add/products/candidates` — 获取可自动添加到促销活动中的商品列表
- `POST` `/v1/actions/auto-add/products/delete` — 从促销活动自动添加列表中删除商品
- `POST` `/v1/actions/auto-add/products/update` — 在促销活动自动添加列表中添加或更新商品

## Ozon促销活动

### `GET` `/v1/actions`

**活动清单**

`operationId: Promos`

用于获取可参与的 Ozon 促销活动列表。  [了解更多关于 Ozon 促销活动的信息](https://global-help.ozon.com/zh/promotion/big-promotions/rasprodazha/)

#### 响应（`seller_apiGetSellerActionsV1Response`）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | number | 否 | 活动识别号。 |
| title | string | 否 | 活动名称。 |
| action_type | string | 否 | 活动类型。 |
| description | string | 否 | 活动描述。 |
| date_start | string | 否 | 活动开始日期。 |
| date_end | string | 否 | 活动结束日期。 |
| auto_add_dates | array<string> | 否 | 商品自动添加到促销活动中的日期和时间。 |
| freeze_date | string | 否 | 活动暂停的日期。  如果该空白被填写，卖家就不能提高价格，改变商品清单或减少促销活动的单位数量。  卖方可以降低价格，增加促销的单位数量。 |
| potential_products_count | number | 否 | 可供活动的商品数量。 |
| participating_products_count | number | 否 | 参加促销的商品数量。 |
| is_participating | boolean | 否 | 无论你是否参加这项活动。 |
| is_voucher_action | boolean | 否 | 此迹象表明买家需要促销代码才能参加。 |
| banned_products_count | number | 否 | 被封商品数量。 |
| with_targeting | boolean | 否 | 此迹象表明该活动是与目标受众一起进行的。 |
| order_amount | number | 否 | 预定金额。 |
| discount_type | string | 否 | 折扣类型。 |
| discount_value | number | 否 | 折扣力度。 |

### `POST` `/v1/actions/candidates`

**可用的促销商品清单**

`operationId: PromosCandidates`

通过识别号获取可参与促销活动的商品清单的方法。

#### 请求体（`seller_apiGetSellerProductV1Request`）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| action_id | number | 否 | 活动识别号。可以使用方法 [/v1/actions](#operation/Promos)获取。 |
| limit | number | 否 | 每页的答复数量。在默认情况下 — 100。 |
| last_id | number | 否 | 页面上最后一个值的ID。运行第一个查询时，将此字段留空。 |

#### 响应（`seller_apiGetSellerProductV1Response`）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| products | array<object> | 否 | 商品清单。 |
|   products[].id | number | 否 | 商品识别号。 |
|   products[].price | number | 否 | 不含折扣的商品的当前价格。 |
|   products[].action_price | number | 否 | 促销价格。 |
|   products[].alert_max_action_price_failed | boolean | 否 | 如果商品价格高于建议价，则为`true`。商品被标记为红色，可能会被排除在促销活动之外。 |
|   products[].alert_max_action_price | number | 否 | 促销商品建议价格。 |
|   products[].max_action_price | number | 否 | 可能的最高促销价格。 |
|   products[].add_mode | string | 否 | 添加到促销中的商品类型：自动或由卖家手动添加。 |
|   products[].min_stock | number | 否 | 库存折扣促销中的最小商品数。 |
|   products[].stock | number | 否 | 《库存折扣》促销中的商品单位数量。 |
|   products[].current_boost | number | 否 | 商品提升幅度。 |
|   products[].price_min_elastic | number | 否 | 最小提升幅度对应的商品价格。 |
|   products[].price_max_elastic | number | 否 | 最大提升幅度对应的商品价格。 |
|   products[].min_boost | number | 否 | 最小提升幅度（%）。 |
|   products[].max_boost | number | 否 | 最大提升幅度（%）。 |
| total | number | 否 | 可用于活动的商品总数。 |
| last_id | number | 否 | 页面上最后一个值的ID。运行第一个查询时，将此字段留空。  要检索以下数值，请从上一个查询的响应中指定`last_id`。 |
