## 商品价格和库存

### `POST` `/v2/products/stocks`

**更新库存商品的数量**

`operationId: ProductAPI_ProductsStocksV2`

可以改变一个商品的库存数量信息。  转交库存数量是当前可用库存，不包括已预留库存。在更新库存之前，请使用以下方法检查已预留库存数量：/v1/product/info/stocks-by-warehouse/fbs。  在一个请求中可以修改100 组商品-仓库的库存。从一个卖家账户最多每分钟可以发送80 个请求。  每30秒内只能为一组商品-仓库更新一次库存，否则在回复中的result.errors参数中将出现TOO_MANY_REQUESTS错误。  只有当一个商品的状态改变为 `price_sent` 时，才有可能设置其可用性。  大件商品的库存只能在其指定的库存地点更新。  如果请求中同时包含参数 `offer_id` 和 `product_id`，系统将优先根据 `offer_id` 应用更改。为避免歧义，建议仅使用一个参数。

#### 请求体（`productv2ProductsStocksRequest`）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| stocks | array<object> | 是 | 仓库中商品的信息。 |
|   stocks[].offer_id | string | 否 | 卖家系统中的商品编号是 — 商品代码。 |
|   stocks[].product_id | integer | 是 | Ozon系统中商品的标识符 — `product_id`。 |
|   stocks[].stock | integer | 是 | 扣除预留库存后的可售商品数量。 |
|   stocks[].warehouse_id | integer | 是 | 得出的仓库编号的方法 [/v1/warehouse/list](#operation/WarehouseAPI_WarehouseList)。 |

#### 响应（`productv2ProductsStocksResponse`）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| errors | array<object> | 否 | 在搜索处理过程中发生的数组错误。 |
|   errors[].code | string | 否 | 错误代码。 |
|   errors[].message | string | 否 | 错误原因。 |
| offer_id | string | 否 | 卖家系统中的商品编号是 — 商品代码。 |
| product_id | integer | 否 | Ozon系统中商品的标识符 — `product_id`。 |
| updated | boolean | 否 | 如果商品信息已被成功更新 — `true`。 |
| warehouse_id | integer | 否 | 仓库编号。 |
