## 与买家的聊天

### `POST` `/v1/chat/send/message`

**发送信息**

`operationId: ChatAPI_ChatSendMessage`

仅适用于拥有 Premium Plus 或 Premium Pro 订阅的卖家。  通过识别码向现有聊天发送消息。

#### 请求体（`chatChatSendMessageRequest`）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| chat_id | string | 是 | 聊天识别码。 |
| text | string | 是 | plain文本格式的信息文本1到1000个字符。 |

#### 响应（`chatChatSendMessageResponse`）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| result | string | 否 | 请求的处理结果。 |

#### 响应（`chatChatSendFileResponse`）

| 字段名 | 类型 | 必填 | 说明 |

### `POST` `/v3/chat/list`

**聊天清单**

`operationId: ChatAPI_ChatListV3`

根据指定的过滤器发回关于聊天的信息。

#### 请求体（`v3ChatList`）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| filter | object | 否 | 按聊天过滤。 |
|   filter.chat_status | string | 否 | 按聊天状态过滤： - `All` — 所有聊天。 - `Opened` — 开放的聊天。 - `Closed` — 不开放的聊天。  默认值：`All`。 |
|   filter.unread_only | boolean | 否 | 按有未读信息的聊天过滤。 |
| limit | integer | 是 | 回答中值的数量。默认值为30。最大值是100。 |
| cursor | string | 否 | 后续数据的选择标志。 |

#### 响应（`v3ChatListResponse`）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| chats |  | 否 | 聊天数据。 |
| total_unread_count | integer | 否 | 未读信息总数。 |
| cursor | string | 否 | 后续数据的选择标志。 |
| has_next | boolean | 否 | 表示响应中未包含全部聊天:   - `true` — 使用新的 `cursor` 参数重新发送请求以获取剩余聊天; - `false` — 响应中已包含匹配请求筛选器的所有聊天。 |

### `POST` `/v3/chat/history`

**聊天历史记录**

`operationId: ChatAPI_ChatHistoryV3`

恢复聊天室消息历史记录。默认顺序为从最新消息到之前的消息。

#### 请求体（`v3ChatHistoryRequest`）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| chat_id | string | 是 | 聊天识别码。 |
| direction | string | 否 | 信息排序方向： - `Forward` — 从旧到新。 - `Backward` — 从新到旧。  默认值是 — `Backward`。消息的数量可以在 `limit`参数中设置。 |
| filter | object | 否 | 消息过滤器。 |
|   filter.message_ids | array<string> | 否 | 消息标识符。 |
| from_message_id | integer | 否 | 从该信息开始整理聊天记录的消息识别码。默认为从最后一条可见信息。  当 `direction = Forward` 时，`from_message_id` 参数为必填。 |
| limit | integer | 否 | 答复的信息数量。默认设置为50。最大值是1000。 |

#### 响应（`v3ChatHistoryResponse`）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| has_next | boolean | 否 | 表示不是所有信息都在答复中返回。 |
| messages | array<object> | 否 | 根据请求正文中的`direction`参数排序的信息数组。 |
|   messages[].context | object | 否 | 聊天室信息。 |
|     messages[].context.order_number | string | 否 | 订单编号。 |
|     messages[].context.sku | string | 否 | Ozon系统中的商品识别码是SKU。 |
|   messages[].created_at | string | 否 | 信息创建日期。 |
|   messages[].data | array<string> | 否 | Markdown格式的带有信息内容的数组。 |
|   messages[].is_image | boolean | 否 | 消息包含图片的标志。 |
|   messages[].is_read | boolean | 否 | 表示信息已读。 |
|   messages[].messageId | integer | 否 | 信息识别码。 |
|   messages[].moderate_image_status | string | 否 | 图片审核状态：   - `SUCCESS` — 审核成功了;   - `MODERATION` — 审核中;   - `FAILED` — 审核失败了. |
|   messages[].user | object | 否 | 关于聊天参与者的信息。 |
|     messages[].user.id | string | 否 | 聊天参与者的身份。 |
|     messages[].user.type | string | 否 | 聊天参与者类型： - `Customer` — 买家， - `Seller` — 卖家， - `Crm` — 系统信息， - `Courier` — 快递员， - `Support` — 客服， - `NotificationUser` — 通知。 |

|   arrival_passes[].arrival_reasons | array<string> | 否 | 入场目的。 |
|   arrival_passes[].arrival_time | string | 否 | 入场日期和时间，UTC格式。 |

|   arrival_passes[].driver_phone | string | 否 | 司机的电话号码。 |
|   arrival_passes[].dropoff_point_id | integer | 否 | 发运点ID。 |
|   arrival_passes[].is_active | boolean | 否 | 如果申请是活跃的，则为`true`。 |
|   arrival_passes[].vehicle_license_plate | string | 否 | 车辆牌照号码。 |
|   arrival_passes[].vehicle_model | string | 否 | 车辆型号。 |
|   arrival_passes[].warehouse_id | integer | 否 | 卖家仓库ID。 |
| cursor | string | 否 | 用于获取下一批数据的指针。 如果此参数为空，则没有更多数据。 |
