# 退货与取消

> Ozon Seller API — 退货与取消
>
> 基础地址：`https://api-seller.ozon.ru`
>
> 认证请求头：`Client-Id`、`Api-Key`、`Content-Type: application/json`

---

## 目录

**FBO和FBS商品退货**（1）

- `POST` `/v1/returns/list` — FBO和FBS退货信息

**rFBS商品退货**（8）

- `POST` `/v2/returns/rfbs/list` — 退货申请列表
- `POST` `/v2/returns/rfbs/get` — 退货申请信息
- `POST` `/v2/returns/rfbs/reject` — 拒绝退货申请
- `POST` `/v2/returns/rfbs/compensate` — 退还部分商品金额
- `POST` `/v2/returns/rfbs/verify` — 批准退货申请
- `POST` `/v2/returns/rfbs/receive-return` — 确认收到待检查商品
- `POST` `/v2/returns/rfbs/return-money` — 向买家退款
- `POST` `/v1/returns/rfbs/action/set` — 传递 rFBS  退货的可用操作

**取消订单**（3）

- `POST` `/v2/conditional-cancellation/list` — 获取 rFBS 取消申请列表
- `POST` `/v2/conditional-cancellation/approve` — 确认 rFBS 取消申请
- `POST` `/v2/conditional-cancellation/reject` — 拒绝 rFBS 取消申请

### `POST` `/v2/returns/rfbs/reject`

`operationId: RFBSReturnsAPI_ReturnsRfbsRejectV2`

### `POST` `/v2/conditional-cancellation/approve`

**确认 rFBS 取消申请**

`operationId: CancellationAPI_ConditionalCancellationApproveV2`

此方法可将状态为 `ON_APPROVAL` 的取消申请标记为已确认。订单将被取消，款项退还给买家。

---

### `POST` `/v2/conditional-cancellation/reject`

**拒绝 rFBS 取消申请**

`operationId: CancellationAPI_ConditionalCancellationRejectV2`

此方法可拒绝状态为 `ON_APPROVAL` 的取消申请。在 `comment` 参数中说明拒绝原因。订单将保留当前状态，并需继续发货给买家。
